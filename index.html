数据分析流程与相关技术

Posted on 2018-10-17 | Edited on 2018-10-18

数据科学中一项任务的流程通常由以下5步组成：

  1. 获取数据
  2. 数据预处理
  3. 数据分析
  4. 分析报告
  5. 化洞见为行动

下面简要介绍整个过程以及其中涉及到的主要技术或工具。

获取数据(Acquire Data)

获取数据的过程主要涉及以下任务：

  • 寻找数据集
  • 提取数据
  • 查询数据

数据的来源非常之多，访问的方式各不相同，下面列举几个常见的来源：

  • 传统的数据库 - 如 MySQL, Oracle等，通常使用 SQL 进行查询。
  • 文本文件 - 如 CSV, JSON 等，可使用脚本语言处理。
  • 远程数据 - 通过 Web 服务进行访问。
  • NoSQL 存储 - 如 Redis, MongoDB, HBase 等，通过其 API 或 Web 服务访问。

准备数据(Prepare Data)

获取数据之后，首先需要探索数据以形成对现有数据的理解，随后对数据进行必要的预处理，为下一阶段的数据分析做准备。

scikit-learn 笔记 - 模型选择与评估

Posted on 2018-10-16 | Edited on 2018-10-17

性能评估

为了防止模型过拟合训练数据，需要从数据集划分一部分数据到测试集，在独立的测试集上评估性能（泛化能力）。

scikit-learn 提供了多种评估方法，包括：

  • 学习器的 score 方法返回默认的评估指标。
  • 使用交叉验证的模型评估工具内部使用了相应的评分策略
  • metrics 模块实现了用于各种不同目的的性能评估指标。

交叉验证 (Cross Validation)

在评估不同的模型参数(超参数)的效果时，直接使用测试集将有可能存在过拟合测试集的风险（多次人为调整模型参数再用测试集验证择优，相当于让模型某种程度学习了测试集），此时在测试集上评估泛化性能就不再准确。

我们可以选择将数据集划分为训练集、验证集、测试集，在验证集上调试模型参数，但这样每个集合的样本量就会减少，在总的样本数较少时，就不太合适。

  2. 分配：根据离centroid的距离将样本归属到某一聚类
  3. 优化：根据聚类所属样本重新计算centroid
